## 一、MVP 定义

### 1.1 MVP 核心范围

**MVP（Minimum Viable Product，最小可行产品）原则：**
- 只做核心功能，解决最重要的问题
- 快速上线，收集反馈，快速迭代
- 不追求完美，追求可用

### 1.2 MVP 功能清单

| # | 功能模块 | 功能点 | 优先级 | 预计工时 |
|---|---------|--------|--------|---------|
| **1. 基础框架** ||||
| 1.1 | 项目初始化 | 技术栈搭建、代码规范、Git 仓库 | P0 | 1天 |
| 1.2 | 用户认证 | 登录、注册、登出、JWT | P0 | 2天 |
| 1.3 | 权限管理 | 角色定义、权限分配 | P0 | 2天 |
| 1.4 | 通知系统 | 站内通知 | P1 | 1天 |
| **2. 需求管理** ||||
| 2.1 | 需求池 | 需求列表、筛选、搜索 | P0 | 2天 |
| 2.2 | 需求 CRUD | 创建、编辑、删除、详情 | P0 | 2天 |
| 2.3 | 需求状态 | 状态流转、状态变更记录 | P0 | 1天 |
| 2.4 | 需求看板 | 看板视图、拖拽 | P0 | 2天 |
| **3. 项目管理** ||||
| 3.1 | 项目列表 | 项目列表、筛选、搜索 | P0 | 1天 |
| 3.2 | 项目详情 | 项目概览、成员、关联需求 | P0 | 2天 |
| 3.3 | 任务看板 | 看板视图、拖拽 | P0 | 2天 |
| 3.4 | 任务 CRUD | 创建、编辑、删除、详情 | P0 | 2天 |
| 3.5 | Sprint | Sprint 创建、开始、结束 | P0 | 2天 |
| 3.6 | 燃尽图 | Sprint 燃尽图 | P1 | 1天 |
| **4. 评审管理** ||||
| 4.1 | 评审列表 | 评审列表、筛选 | P0 | 1天 |
| 4.2 | CDCP 评审 | 概念→计划评审流程 | P0 | 2天 |
| 4.3 | PDCP 评审 | 计划→开发评审流程 | P0 | 2天 |
| 4.4 | ADCP 评审 | 发布评审流程 | P0 | 2天 |
| **5. 仪表盘** ||||
| 5.1 | 项目总览 | 卡片统计 | P0 | 1天 |
| 5.2 | 项目进度 | 进度可视化 | P0 | 1天 |

**总工时估算：约 32 人天**

## 二、开发阶段划分

### 2.1 Sprint 规划

```
Sprint 0: 准备阶段（1天）
├── 项目初始化
├── Git 仓库创建
├── 开发规范制定
└── 需求澄清会议

Sprint 1: 基础框架（5天）
├── 用户认证
├── 权限管理
└── 通知系统

Sprint 2: 需求管理（7天）
├── 需求池
├── 需求 CRUD
├── 需求状态流转
└── 需求看板

Sprint 3: 项目管理（7天）
├── 项目列表+详情
├── 任务看板
├── Sprint 管理
└── 燃尽图

Sprint 4: 评审管理（5天）
├── 评审列表
├── CDCP/PDCP/ADCP 评审
└── 评审流程

Sprint 5: 仪表盘+优化（5天）
├── 项目总览
├── 进度可视化
├── Bug 修复
└── 性能优化

Sprint 6: 测试+发布（5天）
├── 集成测试
├── UAT 测试
├── 部署上线
└── 用户培训
```

**总周期：约 6 周（30 个工作日）**

## 三、技术债务管控

### 3.1 MVP 阶段允许的技术债务

| 类型 | 说明 | 处理时机 |
|------|------|---------|
| 代码注释 | MVP 阶段可适当减少注释 | 后续重构 |
| 测试覆盖 | MVP 覆盖率目标 > 50% | 后续提升 |
| 错误处理 | 基础错误处理即可 | 后续完善 |
| 移动端 | MVP 专注 PC 端 | v2.0 |

### 3.2 MVP 阶段禁止的技术债务

- ❌ 安全性漏洞
- ❌ SQL 注入风险
- ❌ 数据丢失风险
- ❌ 关键功能缺失

## 四、里程碑定义

```
里程碑 1（M1）：基础框架完成
时间：Sprint 1 结束时
标准：登录、注册、权限基础可用

里程碑 2（M2）：需求+项目管理完成
时间：Sprint 3 结束时
标准：核心业务流程跑通

里程碑 3（M3）：评审管理完成
时间：Sprint 4 结束时
标准：IPD 评审流程完整可用

里程碑 4（M4）：MVP 发布
时间：Sprint 6 结束时
标准：功能可用、bug 修复、性能达标

里程碑 5（M5）：正式上线
时间：M4 后 1 周
标准：生产环境稳定、文档完善
```
