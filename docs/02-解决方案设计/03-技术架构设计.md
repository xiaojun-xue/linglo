# 解决方案设计（三）：技术架构设计

> 项目：IPD-ProjectPlatform | 版本：v1.0 | 日期：2026-04-10
> 定位：面向中小企业的轻量级 IPD 研发管理平台

---

## 一、技术选型原则

### 1.1 选型哲学

| 原则 | 说明 |
|------|------|
| **简单优先** | 用最简单的方式解决问题，不过度设计 |
| **工程化友好** | 便于团队协作、代码维护、持续迭代 |
| **中小企业友好** | 部署简单、运维成本低、资源占用少 |
| **信创兼容** | 支持国产数据库、操作系统 |
| **可扩展** | 架构分层清晰，便于后续功能扩展 |

### 1.2 技术栈选择

**核心原则：主流、成熟、社区活跃**

| 层级 | 技术选型 | 备选方案 | 选型理由 |
|------|---------|---------|---------|
| **前端** | Vue 3 + Vite + TypeScript | React | 学习曲线低、团队熟悉、生态好 |
| **UI 框架** | Element Plus | Ant Design Vue | 组件丰富、中文文档优秀 |
| **后端** | Spring Boot 3 + Java 17 | — | 生态成熟、企业级、稳定 |
| **数据库** | PostgreSQL | MySQL | 功能强大、JSON 支持好、信创友好 |
| **缓存** | Redis | — | 缓存 + Session 共享 |
| **消息队列** | — | Kafka（可选）| MVP 阶段暂不需要 |
| **文件存储** | MinIO（对象存储）| 阿里云 OSS | S3 兼容、私有化部署 |
| **工作流** | Flowable | Camunda | 开源、功能完整、活跃 |
| **搜索** | Elasticsearch | — | 如需全文搜索可扩展 |
| **部署** | Docker + Docker Compose | K8S（大型部署）| 简单轻量 |
| **CI/CD** | GitHub Actions / Jenkins | — | 自动化构建部署 |

### 3.2 核心数据表设计

#### 3.2.1 用户与权限

```sql
-- 用户表
CREATE TABLE sys_user (
    id              BIGSERIAL PRIMARY KEY,
    username        VARCHAR(50) NOT NULL UNIQUE,
    password        VARCHAR(255) NOT NULL,
    nickname        VARCHAR(100),
    email           VARCHAR(100),
    phone           VARCHAR(20),
    avatar          VARCHAR(500),
    status          SMALLINT DEFAULT 1,  -- 1:正常 0:禁用
    last_login_time TIMESTAMP,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 角色表
CREATE TABLE sys_role (
    id          BIGSERIAL PRIMARY KEY,
    role_code   VARCHAR(50) NOT NULL UNIQUE,
    role_name   VARCHAR(100) NOT NULL,
    description VARCHAR(500),
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户角色关联表
CREATE TABLE sys_user_role (
    user_id BIGINT,
    role_id BIGINT,
    PRIMARY KEY (user_id, role_id)
);

-- 权限表
CREATE TABLE sys_permission (
    id          BIGSERIAL PRIMARY KEY,
    parent_id   BIGINT,
    name        VARCHAR(100) NOT NULL,
    code        VARCHAR(100) NOT NULL UNIQUE,
    type        SMALLINT,  -- 1:菜单 2:按钮 3:API
    path        VARCHAR(200),
    component   VARCHAR(200),
    sort_order  INT DEFAULT 0,
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 角色权限关联表
CREATE TABLE sys_role_permission (
    role_id       BIGINT,
    permission_id BIGINT,
    PRIMARY KEY (role_id, permission_id)
);
```

## 四、API 接口设计

### 4.1 RESTful API 规范

**基础路径：** `/api/v1`

**认证方式：** JWT Token（Bearer Token）

**请求规范：**
```
Content-Type: application/json
Authorization: Bearer {token}
```

**响应格式：**
```json
{
  "code": 200,           // 业务状态码
  "message": "success",  // 消息
  "data": { ... },       // 数据
  "timestamp": 1712700000
}
```

## 六、部署架构

### 6.1 Docker Compose 部署（推荐中小企业）

```yaml
# docker-compose.yml
version: '3.8'

services:
  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
      - ./frontend/dist:/usr/share/nginx/html
    depends_on:
      - backend
    networks:
      - ipd-network

  # 后端服务
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=postgres
      - DB_PORT=5432
      - REDIS_HOST=redis
    depends_on:
      - postgres
      - redis
    networks:
      - ipd-network

  # 数据库
  postgres:
    image: postgres:16-alpine
    environment:
      - POSTGRES_DB=ipd_platform
      - POSTGRES_USER=ipd_user
      - POSTGRES_PASSWORD=ipd_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - ipd-network

  # Redis 缓存
  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    networks:
      - ipd-network

  # MinIO 对象存储
  minio:
    image: minio/minio:latest
    command: server /data --console-address ":9001"
    environment:
      - MINIO_ROOT_USER=minioadmin
      - MINIO_ROOT_PASSWORD=minioadmin
    volumes:
      - minio_data:/data
    networks:
      - ipd-network

volumes:
  postgres_data:
  redis_data:
  minio_data:

networks:
  ipd-network:
    driver: bridge
```

### 6.2 环境配置
