### 2.1 数据隔离策略

**测试前：** 每次测试套件启动时创建干净的 H2 数据库 schema（`ddl-auto: create-drop`）

**测试中：** 每个测试方法使用独立 `@Transactional` + `rollback=true`，测试结束后自动回滚

**测试后：** Jenkins/CI 定时清理策略
- E2E 测试数据：测试完成后执行 `DELETE` 清理脚本
- 视觉测试截图：`git clean` 或截图命名按日期归档

## 三、增量测试策略（代码修改后）

### 3.1 Git Hook 自动触发

```bash
# .git/hooks/post-commit
#!/bin/bash
CHANGED_FILES=$(git diff --name-only HEAD~1)

# 如果修改了后端代码 → 运行后端测试
if echo "$CHANGED_FILES" | grep -q "src/backend"; then
    echo "Backend changed → running backend tests..."
    cd src/backend && mvn test
fi

# 如果修改了前端代码 → 运行前端 E2E 测试
if echo "$CHANGED_FILES" | grep -q "src/frontend"; then
    echo "Frontend changed → running E2E tests..."
    cd src/frontend && pnpm exec playwright test
fi

# 如果修改了 API 接口定义 → 运行契约测试
if echo "$CHANGED_FILES" | grep -q "src/backend/controller"; then
    echo "API changed → running API tests..."
    cd src/backend && mvn test -Dtest="*Controller*Test"
fi
```

### 3.2 变更影响分析矩阵

| 修改范围 | 自动触发测试 |
|---------|------------|
| `entity/*.java` | 全量测试（数据结构变更影响所有模块） |
| `controller/*.java` | API 测试 + E2E 测试 |
| `service/*.java` | Service 测试 + E2E 测试 |
| `views/*.vue` | E2E 测试 + 视觉回归 |
| `api/*.ts` | E2E 测试 |
| `router/*.ts` | E2E 全流程测试 |
| `**/*.css` | 视觉回归测试 |
| `pom.xml` / `package.json` | 全量测试 |

### 4.3 视觉测试配置

```typescript
// src/frontend/tests/visual.spec.ts
import { test, expect } from '@playwright/test';
import { setupScreenshotEnv, compareScreenshots } from './utils/visual';

const pages = [
  { name: 'login', path: '/login', waitFor: 'h1' },
  { name: 'dashboard', path: '/dashboard', waitFor: '.stat-cards' },
  { name: 'requirements', path: '/requirements', waitFor: '.filter-bar' },
  { name: 'projects', path: '/projects', waitFor: '.content-card' },
  { name: 'tasks', path: '/tasks', waitFor: '.task-board' },
  { name: 'sprints', path: '/sprints', waitFor: '.sprint-list' },
  { name: 'reviews', path: '/reviews', waitFor: '.review-table' },
  { name: 'qa', path: '/qa', waitFor: '.qa-tabs' },
  { name: 'docs', path: '/docs', waitFor: '.doc-tree' },
];

for (const page of pages) {
  test(`[Visual] ${page.name} - layout regression`, async ({ page: p }) => {
    await setupScreenshotEnv(p, page.name);
    await p.goto(`http://localhost:3000${page.path}`);
    await p.waitForSelector(page.waitFor, { timeout: 10000 });
    await p.waitForTimeout(1000); // 等待动画完成

    const screenshot = await p.screenshot({ fullPage: false });
    await compareScreenshots(page.name, screenshot);
  });
}
```

### 5.1 测试执行时机

| 触发方式 | 执行内容 | 预计时长 |
|---------|---------|---------|
| **代码提交时** (pre-commit hook) | 增量测试（改动影响范围） | ~5 分钟 |

### 6.2 前端 E2E 测试数据清理

```typescript
// tests/utils/cleanup.ts
export async function cleanupTestData() {
  // E2E 测试使用统一前缀，避免污染真实数据
  const testPrefix = `e2e_${Date.now()}_`;

  // 测试后删除以 testPrefix 开头的所有数据
  await request.delete(`/api/requirements?createdByStartsWith=${testPrefix}`);
  await request.delete(`/api/projects?createdByStartsWith=${testPrefix}`);
  await request.delete(`/api/tasks?createdByStartsWith=${testPrefix}`);
  await request.delete(`/api/users?usernameStartsWith=${testPrefix}`);
}

// 每个测试文件afterAll钩子调用
afterAll(async () => {
  await cleanupTestData();
});
```

## 七、技术选型与依赖

| 组件 | 选型 | 理由 |
|------|------|------|
| 后端测试框架 | JUnit 5 + Spring Boot Test | Spring 官方推荐，原生集成 |
| API 测试 | RestAssured / MockMvc | 简洁的 DSL，支持 Spring Test |
| 前端 E2E | Playwright + TypeScript | 支持所有浏览器、截图强大、社区活跃 |
| 视觉对比 | Playwright screenshot + pixelmatch | 无需额外付费，集成在 Playwright 中 |
| 测试报告 | Allure Report | 丰富的可视化报告，支持 CI 集成 |
| CI/CD | Jenkins / GitHub Actions | 均可，支持定时触发 |
| 测试数据 | H2 内存数据库 + @Transactional | 零污染，速度快 |
