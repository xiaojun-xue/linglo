## 三、分析维度框架

### 3.1 产品功能维度

| 维度 | 说明 |
|------|------|
| IPD 流程支持 | 是否支持 IPD 流程定制、门禁评审 |
| 项目模板 | 预设模板丰富度、行业适配性 |
| 敏捷/Scrum | Sprint、Backlog、燃尽图等 |
| 看板管理 | 看板自定义、自动化流转 |
| 需求管理 | 需求池、需求变更追踪 |
| 任务管理 | 任务分解、工时预估、依赖关系 |
| 测试管理 | 测试用例、缺陷管理 |
| 报表与度量 | 项目健康度、效率指标、数据可视化 |
| 文档管理 | 与文档系统集成能力 |

### 3.2 技术架构维度

| 维度 | 说明 |
|------|------|
| 部署方式 | SaaS / 私有化 / 混合部署 |
| 技术栈 | 前端/后端/数据库选型 |
| 扩展性 | 插件机制、API 开放性 |
| 集成能力 | 与 Git、CI/CD、IM、邮件等工具集成 |
| 性能与稳定性 | 并发用户支持、数据容量 |

### 3.3 市场与用户体验维度

| 维度 | 说明 |
|------|------|
| 目标用户 | 企业规模、行业类型 |
| 定价策略 | 按用户/项目/功能分级 |
| 学习成本 | 上手难度、培训需求 |
| 客户案例 | 行业标杆客户 |
| 市场份额 | 市场认知度 |
| 客户服务 | 技术支持、SLA |

### 3.4 IPD 特色维度（差异化重点）

| 维度 | 说明 |
|------|------|
| IPD 阶段门 | Gate Review、决策评审点 |
| 业务对齐 | 市场需求到技术实现的端到端追踪 |
| 跨部门协同 | 市场、研发、服务高效协同 |
| 投资组合管理 | 多项目优先级排序与资源平衡 |
| 生命周期管理 | 产品从概念到退市的全周期管理 |
